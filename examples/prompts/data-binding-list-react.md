# Data Binding List In React

## Prompt

```text
How should I make a dynamic list in Rive and update it from React? Should I use Inputs?
```

## Good Answer Should Mention

- Data Binding with View Models is the modern path for data-driven UI.
- Legacy Inputs are deprecated for new data-driven workflows.
- Lists should be modeled in the Rive file and populated through runtime data binding.
- React-specific code needs current source verification before exact API signatures are provided.
- Useful local references include [rive-reference/03-data-binding.md](../../rive-reference/03-data-binding.md), [rive-reference/07-web-react-runtime.md](../../rive-reference/07-web-react-runtime.md), and [rive-recipes/data-driven-list.md](../../rive-recipes/data-driven-list.md).

## Red Flags

- Recommends Legacy Inputs as the default solution.
- Skips View Models and Data Binding entirely.
- Gives confident exact React runtime API calls without source verification.
