# Contributing

Thanks for helping improve rive-assistant. Contributions fall into a few categories — here's how each one works.

## Updating reference files

The `rive-reference/` files are hand-curated summaries of Rive's documentation. Each file covers a domain (editor, state machines, scripting, etc.) and is intentionally concise — long enough to be useful, short enough to fit in an AI context window.

When editing:
- Match the existing tone and formatting
- Keep examples minimal and illustrative
- Prefer condensing over expanding — remove redundancy rather than adding more prose
- Source information from the [official Rive docs](https://github.com/rive-app/rive-docs)

## Updating the routing table in `CLAUDE.md`

`CLAUDE.md` contains a routing table that maps Rive topics to specific documentation paths. If you add a new reference file or the Rive team reorganizes their docs, the routing table needs updating too.

The `docs-paths.txt` file is auto-generated by the sync workflow — **do not edit it manually**. If you need to regenerate it, trigger the workflow manually from the Actions tab.

## Updating the rive-docs submodule

The `rive-docs/` submodule is the full Rive documentation for offline use. To update it:

```bash
git submodule update --remote
```

Don't commit submodule pointer bumps as standalone changes — include them alongside related reference file updates.

## Opening issues

Use the issue templates. For incorrect or outdated information, include a link to the authoritative source so the fix is unambiguous.

## Pull requests

Use the PR template. The key question for almost every PR: does `CLAUDE.md`'s routing table need updating?
